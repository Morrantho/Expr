log.c:
* Added basic logging. It manages 2 arrays. An AOB store / arena where fully
formatted logs are stored, and an entry vector for the entries. It
fully formats the log format string + args first, then pushes to
the AOB, which returns the AOB offset. Then we push an entry, write the
src file name, current line and column, log type, and AOB offset. We
need those for later flushing. Flushing loops all entries, gets the
formatted log from the AOB, and writes all the stored data to printf.
We can add log level colors later. For now all we need is WARN and FATAL.
Idk if Flushing is the term to use here, I'm just writing what I think
makes sense. I've never written one of these, nor needed to, but we need
one, so here it is.

app.c:
* Implemented logger. Calls LogInit, LogFree, and LogReset where necessary.
We call LogFlush to display all logs, then check for fatal errors, which
prevents executing bad code.
